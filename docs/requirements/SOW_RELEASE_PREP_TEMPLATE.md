# リリース準備SOW（SOW_RELEASE_PREP）テンプレート

このファイルは **「要件定義時に未定だが、リリースまでに必須」** とされた項目（アセット、文言、契約等）を管理するためのSOWテンプレートです。
`docs/requirements/projects/<project_slug>/SOW_RELEASE_PREP.md` として保存し、通常の開発SOWと並行して消化します。

---

## 1. 基本情報
- プロジェクト: <project_slug>
- 目的: リリースに向けた「開発以外の」準備事項の完遂
- 期限: **リリース1週間前まで（厳守）**

## 2. 必須TODOリスト（未決定事項の解消）

**※REQUIREMENTS.md の「リリースまでの必達TODO」を転記・具体化**

### 2.1 メタ情報・文言
- [ ] サイトタイトル（title）の最終決定
    - 決定内容:
- [ ] サイト説明文（description）の最終決定
    - 決定内容:
- [ ] OGP文言/ハッシュタグ等の決定

### 2.2 アセット（素材）
- [ ] ファビコン（favicon.ico / icon.png）の作成と配置
- [ ] Apple Touch Icon の作成と配置
- [ ] OGP画像の作成と配置
- [ ] 404ページ用画像/文言の準備

### 2.3 外部サービス・契約関係
- [ ] ドメイン設定（DNS）の準備
- [ ] Google Search Console / Analytics のプロパティ作成
- [ ] プライバシーポリシー / 利用規約の最終確認（法務確認含む）
- [ ] メール送信プロバイダ（SendGrid等）の検証済みドメイン設定

## 3. 受入基準（Definition of Done）

1. 全てのTODO項目が `[x]` になっていること
2. 決定したテキスト情報（title/description等）が環境変数（`.env`）または定数ファイルに反映されていること
3. 画像アセットが `public/` 等の適切な場所に配置され、表示確認が取れていること

## 4. リスク管理

| 未決項目 | デッドライン | 遅延時の対応（暫定案など） |
|---|---|---|
| 例: ファビコン | ○月○日 | 暫定でロゴを使用 or フリー素材 |
