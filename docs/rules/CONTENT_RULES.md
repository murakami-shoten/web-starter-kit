# コンテンツ可搬性規約（CONTENT_RULES）

（作成日: 2026-01-27 JST）

---

## 1. 方針

- 記事・固定ページ等のコンテンツは **MDX** を第一候補とする
- 目的: CMSロックイン回避 / Gitで履歴管理 / 移植性

---

## 2. ルール（推奨）

- コンテンツは `content/` 配下に置く（例: `content/blog/slug.mdx`）
- frontmatter を必須（title, description, date, updated, tags など）
- 画像は将来S3互換へ移せるように、参照方法を統一する
- 著者/更新日/根拠（出典リンク）をテンプレに含める（SEO信頼性）

---

## 3. 禁止

- CMSにしか存在しない情報を前提にしない（後戻りが難しい）
- build時に外部APIへ依存して記事生成しない（可用性低下）
