# 低ロックイン方針（LOW_LOCKIN_RULES）

（作成日: 2026-01-27 JST）

---

## 1. 「低ロックイン」の現実ライン

完全なロックインゼロは現実的ではないため、以下を方針化する:

- **データ層（DB/Storage）**を最優先で中立にする
- 依存するSaaS/SDKは「差し替え点」を作り、直接参照を広げない
- 監視はベンダー依存が強いので、可能なら OpenTelemetry を前提に“吐き出し口”を共通化する

---

## 2. 具体ルール

### 2.1 DB
- 原則 Postgres を前提（SQL/スキーマは移植しやすい）
- ORM/マイグレーションはGit管理し、環境ごとに再現できること

### 2.2 オブジェクトストレージ
- アプリは「S3互換」的な抽象で扱えるようにする（実体は後で選べる）

### 2.3 認証
- 認証は境界を設ける（Auth provider を切り替えてもアプリ内部が崩れない）
- JWT/セッションなど標準プロトコルを優先する

### 2.4 Observability
- ローカルはconsole、staging/prodはプロバイダ管理画面
- さらに必要になったら OpenTelemetry でログ/メトリクス/トレースを送れる構造にする
